How It Works

opencli 怎麼運作？三層架構

一行 opencli 進來，自動分流走「純 HTTP」或「接你的 Chrome」兩條路

PUBLIC / LOCAL — 不需 Chrome

公開資料（HN 熱榜、天氣、bilibili）走純 HTTP undici fetch，零 LLM、超快。

Building Blocks

opencli + 三條鐵則

opencli browser

沿用既有 Chrome session

不需密碼、不需 WebDriver。透過 Chrome extension 把已登入的分頁變成 AI 可控的 DOM。

  • state：列出帶 [N] 的可點元素
  • click N / type N "text"：用 index 操作
  • eval <JS>：跑 JS（含 cross-frame）
  • screenshot / get / network：抓畫面、值、API 流量
必須遵守

三條鐵則

1. 一條龍跑完
老式 HR 系統閒置 ~2-3 分鐘 session 就失效。中間問問題、等回覆，回來頁面就 about:blank。
2. Bash tool 跑的是 zsh
for x in $VAR 在 zsh 不分詞，批次只跑 1 圈。用 shell function 逐筆呼叫。
3. iframe 內元素點不到
popup grid 的「選取」鈕沒被 state 編 index。用 cross-frame eval + DevExpress client API 繞。

Lessons Learned

踩過的 7 個雷 → 對策

❌ 中間停一下，頁面變 about:blank
整個任務寫成一條 Bash 腳本連續跑完，問問題前先存草稿
❌ for x in $FORMIDS 只跑 1 圈
Bash tool 跑 zsh、不分詞 → 用 shell function 逐筆呼叫
❌ popup grid 的「選取」按鈕沒有 [N]
cross-frame eval：contentDocument.click() 進 iframe 點
❌ 子假別 SetValue 繞過 popup，存檔報「假別額度」
假別編號靠 server callback 帶出 → 一定要透過 popup 點選取
❌ 病假儲存按了沒反應、清單也沒出現
「請假說明」病假必填，無錯誤訊息 → 填「病假」兩字就過
❌ 儲存按鈕 index 從 [50] 飄到 [36]
每次點之前重新 state + grep，別存舊 index 重用
❌ AskUserQuestion 期間 session 失效
問之前先存草稿，問完整批重跑（草稿可逆）
